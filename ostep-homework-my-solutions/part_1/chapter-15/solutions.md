- Run with these flags: -s 0 -n 10. What value do you have set -l (the bounds register) to in order to ensure that all the generated virtual addresses are within bounds?
    - `python3 relocation.py -s 0 -n 10`
        - We must set -l >= 930. More generally, we can set -l >= MAX(size(VA)) + 1, where VA = virtual address and size(VA) is in decimal form. 
        - We can check our answer of 930 using the following command: `python3 relocation.py -s 0 -n 10 -l 930 -c`

- Run with these flags: -s 1 -n 10 -l 100. What is the maximum value that the Bounds (limit) register can be set to, such that the address space still fits into physical memory in its entirety?
    <!-- Read this question as "What is the largest value the LIMIT register can take on before a given process's address space spills outside the boundary of physical memory?" -->
    - Since we have `ARG phys mem size 16k`, physical memory = 16 * 1024 = 16384 bytes.
    - The physical address space occupies a range from BASE to BASE + LIMIT.
    - For it to fit entirely in physical memory, BASE + LIMIT <= 16384.
    - So the maximum LIMIT is 16384 - BASE.
